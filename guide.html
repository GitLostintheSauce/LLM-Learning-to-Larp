The one rule

You learn by interrogating Claude — not by collecting answers. A box gets checked only when you can explain it in your own words, with no AI open. The website you build at the end is the proof, not the goal.

How we keep it honest (open me)
  • Teach-back, every concept. Explain it in your own words before moving on. Can't explain it simply → you don't get it yet.
  • Learning journal. Each session: what I tried to learn, what surprised me, what I had wrong before, one question I still can't answer.
  • Draw it yourself. Hand-draw each phase's diagram, then ask the AI to critique your drawing.
  • Predict before you run. Write what you think will happen, then run it. The gap is the lesson.
  • No-AI checkpoints. Each phase ends with a gate you pass with nothing open.
  • Defend every sentence on your website. If you can't explain it cold, it shouldn't be there.
How to interrogate an LLM so you actually learn
  • "Explain it like I've never heard of it, then add one layer of nuance at a time."
  • "Give me an analogy — then tell me where it breaks down."
  • "What's the layer below this? What's the layer above?"
  • "What am I not asking that I should be asking?"
  • "Quiz me. One question at a time, wait for my answer, then tell me if I'm right and why."
  • "I think it works like X — where am I wrong?"
  • Be skeptical. LLMs make things up confidently. Verify what matters.

what does this actually mean? (open if that felt dense)

You are not building or training a model — Claude already exists; you're just using it. Your program runs a loop: send the conversation to the model → if it replies "use the calculator," your code runs it and sends the answer back → repeat until done.

That loop is all a "harness" like Claude Code really is, minus the polish — building the bare version is what makes it click. Let Claude Code help you write it; the only rule is that you can explain every line. ↳ It's really just you building Phase 1's pillars 3 & 4 by hand.

Run your own open-source model ~3–5 days

what does this mean?

"Locally" means downloading the model's file onto your own computer and using a free app (Ollama or LM Studio) to chat with it — no internet or API needed. It'll feel weaker than Claude; noticing exactly how and why is the point.

what does this mean?

Earlier your homemade agent called Claude over the internet. Change just the part that picks the model so it talks to the model running on your own computer instead — everything else stays the same. That only one piece changes is the whole lesson: the loop (harness) and the brain (model) are separate, swappable parts.

what does this mean?

Fine-tuning means showing an existing model a small set of your own examples so it leans toward the behavior you want — not building a new model. It's optional and curiosity-driven; the real skill is measuring whether it actually improved rather than assuming it did.

Stand up a real agent harness ~4–7 days

what does this mean?

Claude Code is a harness, but its inner recipe is private. An open-source harness is the same kind of tool with its recipe public — so you can read it, run it, and change it, and finally see in full detail how a real harness works.

what does this mean?

First run the open harness on a big model over the internet, so it works well and you learn its settings. Then switch it to the model on your own machine from the step above — the same swap idea as your homemade agent, now on a real tool.
